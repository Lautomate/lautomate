Call Center

Level: Easy

Tools and Techniques: Power Query, Data Cleaning, Data Modeling, Power BI, and Storytelling

Topic: Call Center Performance

NOTE

The dataset used in this project was sourced from Medium by K S Ashish. Although the data was modified, it does not affect the original state of the dataset.

GOAL

The goal of this project is to improve the call center data analysis originally performed in Excel by K S Ashish. It will tackle historical data and provide insights in a clear way.

INTRODUCTION

I have always been fascinated and curious about what it is like to be a data analyst in the BPO industry. I researched possible datasets, questions, and analyses, and found that in-house data is most likely the dataset used in this industry. Fortunately, I came across a Medium post where the author provided a downloadable dataset. The post is considered a case study that explores the data, develops an argument, and offers excellent insights and recommendations.

At the end of this writing, you will see that I developed a Power BI project analyzing the improved data, delivering more impactful insights while demonstrating my abilities to enhance previous projects.

PROCESS

The first step I took was to load the data in Power Query to clean and transform the semi-clean data. I removed unnecessary columns while ensuring that the removed columns did not affect the analysis. I then performed my favorite text cleaning and transformation techniques, namely trimming, cleaning, and replacing double spaces with single space values. The area column contains both the city and its abbreviation; therefore, I replaced it using the replace value technique in one step. I also adjusted the call time column to reflect the relevant year and month for this project.

Before loading the data, I further investigated using the column profiling feature in Power BI to give me an overview of the dataset. One discovery was the granularity of the date, which is daily. To address this, I created a simple date dimension table to cater time-series demands. After this, it was time to load the data and perform the visualization.

The dashboard page displays almost the same metrics as those in the article, with slight tweaks in the design aspects and insights.

LIMITATION

The dataset only contains one month of data, limiting the ability to perform month-on-month or quarterly analyses.

RESULT

The results of this project reveal that values are consistent throughout the month, with little to no changes in the KPIs and metrics. This suggests that management or decision-makers should consider improving strategies or interventions to enhance performance.

The analysis presents information in a clear and understandable manner, using appropriate visuals to either break down or aggregate values. This approach enhances the communication of insights and supports better decision-making, particularly in tracking business performance based on the available customer data.
