Level: Intermediate

Tools and Techniques: Excel, Python, and SQL

Topic: Sales Transactions

NOTE

Please note that this project replicates the actual business scenario but does not use the company's real data.

GOAL

This project aims to reduce data entry errors, enhance business operations, and provides a high-granularity data and modern solution that respects traditional operations.

INTRODUCTION

This project was undertaken to address the need for a modern data management solution that reduces data entry errors and enhances business operations while respecting traditional processes. By leveraging SQL and Python, I developed a project that simplifies data entry, ensures data integrity, and improved the data management workflow. The project involved creating SQL tables for products, shop areas, and transactions, along with an Excel format designed for efficient data entry. A Python application was also developed to automate the process of transferring data from Excel to SQL, minimizing manual effort and reducing the likelihood of errors. The implemented project is designed to accommodate various stakeholders, providing them with accurate and up-to-date information for decision-making.

PROCESS

My first step involved gathering all the requirements and identifying the necessary columns for the project. I created a simple SQL table for products, inserting all 31 products along with their interconnected column values. Following this, I developed a table for shop areas, filling in gaps to provide additional context, such as city, province, region, and island, since the available data only included barangay-level information.

Next, I created a comprehensive transactions table that included all the essential columns, identical to those that I would later implement in Excel. After establishing all the required tables and inputting predefined values, I designed a standardized format in Excel. This format aims to minimize data entry while ensuring data integrity and accuracy. I maximized the use of data validation to streamline the workflow for data entry specialists.

Now, letâ€™s discuss the exciting part: I developed a Python application that reads the Excel file and pushes the data to SQL. I utilized several powerful Python libraries for data handling, database interaction, and packaging. As a result, data entry specialists now only need to input minimal data into Excel, run the standalone Python application, and have the data securely stored in the database.

I recognized that when data is pushed from Excel to SQL, the transactions table may contain incomplete information. This is because I simplify the data entry in Excel and broke down the data into three distinct tables. The first step in my SQL data transformation process involves modifying the Invoice No column values based on the payment method. Since SQL does not interpret custom formats from Excel but rather the actual input, this step is crucial.

The second step is to check and update customer records with multiple age and/or gender entries. Gender should have only one value per unique customer, while age serves as nominal data in the transactions table, answering the question, "How old are you?"

The third step also involves a check and update process where data entry specialists play a significant role. I will identify quantity pattern and outliers in the data and, upon discovery, consult with the data entry specialists to verify and update the values accordingly.

Finally, I performed SQL joins on the transactions table to include all records from the Transactions table along with the matching records from the Products and Areas tables. This data was then organized into views, which will serve as the retrieval point for stakeholders creating reports.

The rationale behind breaking the data into different tables and subsequently joining them is to accommodate various stakeholders engaged in different processes.

LIMITATION

If the user moves or renames the Excel file used for data entry, the Python application will fail to run. This dependency on the specific file path and name could lead to delay in the data entry process if the file is accidentally moved or renamed.

RESULT

This project successfully achieved its goal of reducing data entry errors and enhancing business operations. The development of SQL tables and the standardized Excel format has streamlined the data entry process, making it more efficient and accurate. The Python application has automated the transfer of data from Excel to SQL, significantly reducing the manual effort required and minimizing the risk of errors.

Despite the limitations, the system has proven effective in organizing and managing the data, allowing stakeholders to access reliable information through SQL views for reporting purposes. The project has provided a modern solution that respects traditional operations, ensuring that data entry specialists can continue their work with tools that improve accuracy and efficiency.
